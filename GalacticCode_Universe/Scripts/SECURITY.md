# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| < 0.2   | :x:                |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security vulnerability within this project, please report it responsibly.

### How to Report

1. **Do not** disclose the vulnerability publicly until it has been addressed.
2. Use [GitHub's Security Advisory feature](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability) to privately report the vulnerability through this repository.
3. Include detailed information about the vulnerability:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact
   - Any suggested fixes (optional)

### What to Expect

- **Acknowledgment**: We will acknowledge receipt of your report within 48 hours.
- **Communication**: We will keep you informed about the progress of addressing the vulnerability.
- **Resolution**: We aim to resolve critical vulnerabilities within 7-14 days.
- **Credit**: We will credit you for the discovery (unless you prefer to remain anonymous).

### Scope

This security policy applies to:
- The main repository code
- Dependencies used in the project
- GitHub Actions workflows

Thank you for helping keep this project secure!
